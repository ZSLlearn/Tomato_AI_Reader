# Tomato Reader PRD

## 一、产品概述

### 1.1 产品定位
Tomato Reader 是一款基于 Electron 的桌面端 AI 智能阅读器，采用 IDE 式三栏布局（书架侧边栏 + 阅读区 + AI 对话面板），支持 PDF / DOCX / PPTX / EPUB / TXT 多格式原生预览，集成 DeepSeek 大模型实现选中文字即时解释/翻译、全文上下文对话、引用收集等 AI 辅助阅读功能。

### 1.2 核心目标
- 提供桌面端原生的多格式文档阅读体验
- 通过 AI 三层架构（L1 即时层 / L2 对话层 / L3 异步层）满足不同深度的阅读辅助需求
- 文档管理流程简洁：导入 → 书架归类 → 阅读 → AI 辅助理解

### 1.3 技术栈
- **框架**：Electron 34
- **前端**：原生 HTML / CSS / JS（无框架依赖）
- **PDF 渲染**：PDF.js 5.x
- **DOCX 渲染**：mammoth.js + JSZip
- **AI 接口**：DeepSeek API（兼容 OpenAI 格式）
- **存储**：localStorage（元数据/设置/对话）+ IndexedDB（文档内容/二进制）
- **图标**：Lucide Icons
- **打包**：electron-builder（Windows portable + NSIS 安装包）

### 2.3 AI 辅助功能（三层架构）

#### L1 即时层 — 悬浮窗
| 功能 | 触发方式 | 实现情况 |
|------|----------|----------|
| 引用 (Quote) | 选中文字 → 悬浮窗 💬 按钮 → 添加到引用列表 | ✅ |
| AI 解释 (Explain) | 选中文字 → 悬浮窗 💡 按钮 → 展开解释面板 | ✅ |
| AI 翻译 (Translate) | 选中文字 → 悬浮窗 🔍 按钮 → 展开翻译面板 | ✅ |

悬浮窗行为：
- 选中 ≥ 2 个字符后 200ms 延迟弹出
- 在选中文字上方/下方自适应定位（尖角指向选中区域）
- 解释/翻译面板互斥（同时只展示一个）
- 点击外部 / Esc / 滚动 / 10s 无操作 → 消失

## 四、数据模型

### 4.1 核心实体

```
Document（文档）
  ├─ id, name, type, size, pages
  ├─ shelf（所属书架ID）
  ├─ content（全文文本引用）
  ├─ date（导入日期）
  └─ lastRead（最后阅读日期）

DocumentContent（文档解析内容 — IndexedDB）
  ├─ id
  ├─ pages[]（分页文本数组）
  ├─ html（DOCX/EPUB 渲染 HTML）
  ├─ fullText（全文纯文本）
  ├─ toc[]（目录结构）
  └─ slides（PPTX 幻灯片数据）

ChatSession（对话会话）
  ├─ id
  ├─ documentId（关联文档）
  ├─ title
  ├─ createdAt
  └─ messages[]（消息列表）

Message（消息）
  ├─ role（user / assistant）
  ├─ content
  ├─ timestamp
  ├─ references[]（引用项，可选）
  └─ tokens（token 用量）

QuoteItem（引用项 — 内存存储，不持久化）
  ├─ id
  ├─ type（selection / explanation / translation）
  ├─ text（原始文字）
  ├─ aiContent（AI 结果，可选）
  ├─ pageNumber
  └─ chapterTitle
```

### 4.2 数据存储方案

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 文档元数据 | localStorage | 轻量，快速读写 |
| 文档解析内容 | IndexedDB | 大量文本/HTML，单文档可能数MB |
| 对话历史 | localStorage | JSON 序列化存储 |
| PDF 二进制 | IndexedDB (blob) | 原始文件数据 |
| 引用列表 | 内存 (AppState) | 临时上下文，切换文档/对话清空 |
| 用户设置 | localStorage | 键盘值对 |
| API 配置 | localStorage（混淆） | XOR + Base64 编码 |
| 阅读位置 | localStorage | 每个文档一个键 |

## 五、AI 提示词工程

### 5.1 L1 层（解释/翻译 — 轻量，不携带文档全文）
- **解释** System Prompt：`"你是一个专业的知识解释助手。请用通俗易懂的语言解释用户提供的文字，要求：简洁清晰、逻辑分明..."` + 用户选中的原始文字
- **翻译** System Prompt：`"你是一个专业的翻译助手。请将用户提供的文字翻译为目标语言..."` + 用户选中的原始文字
- 响应目标：1-3 秒

### 5.2 L2 层（对话 — 携带文档全文上下文）
- System Prompt 包含：角色设定 + 文档信息（标题/类型/页数）+ 文档目录结构 + 文档全文内容 + 回答规则
- 用户引用片段以特殊格式注入 User Message：`【用户引用的文档片段】` + 片段列表 + `【用户问题】`
- Token 预算管理：根据文档长度动态分配，超出 40000 字符截断并提示

## 六、安全设计

### 6.1 API Key 存储
- 使用设备指纹（Canvas + Navigator 属性生成）+ XOR 混淆 + Base64 编码
- 存入 localStorage，不在网络传输中明文暴露
- 支持多配置管理，活跃配置独立切换

### 6.2 Electron 安全
- `contextIsolation: true`（上下文隔离）
- `nodeIntegration: false`（禁用 Node 集成）
- 通过 `preload.js` + `contextBridge` 暴露有限 API
- `webSecurity: false`（允许加载本地 PDF.js worker）

## 七、构建与分发

### 7.1 开发模式
```bash
npm start          # 启动本地服务器 (端口 8088)
npm run electron   # 启动 Electron 窗口（加载本地文件）
```
环境变量 `USE_DEV_SERVER=true` 时 Electron 加载开发服务器 URL。

### 7.2 构建
```bash
npm run build:win            # Windows 便携版
npm run build:win:installer  # Windows NSIS 安装包
```
输出目录：`dist-electron/`
